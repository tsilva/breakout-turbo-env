# v0.3.0 macOS ARM64 benchmark

This community-preview measurement compares equivalent policy-facing workloads
on one development machine. It is evidence for this configuration, not a
universal performance guarantee.

## System

- Apple M1 Pro, 8 physical CPU cores, 16 GiB memory
- macOS 26.5.2, ARM64
- Python 3.14.5
- breakout-turbo-env v0.3.0 release candidate
- stable-retro-turbo 1.0.1.post30 at commit `ed214dfff900`

The machine had other desktop workloads running. Five timed repeats and the raw
rates are included so readers can judge variance.

## Matched workload

- 16 environments and 8 worker threads
- 5,000 vector steps per repeat after 1,000 warmup steps
- frame skip 4 and frame stack 4
- grayscale CHW `4×84×84` observations
- area resize, no max-pooling, no info collection, safe-view buffers
- identical repeating noop, FIRE, right, and left native actions
- explicit masked reset for completed lanes

```bash
PYTHONPATH=python ../stable-retro-turbo/.venv/bin/python \
  scripts/benchmark_comparison.py \
  --steps 5000 --warmup 1000 --repeats 5 --threads 8 --json
```

## Results

| Environment | Median env-steps/s | Mean env-steps/s | Sample standard deviation |
| --- | ---: | ---: | ---: |
| breakout-turbo-env | 304,169.5 | 305,240.9 | 15,885.8 |
| stable-retro-turbo | 8,069.2 | 7,873.6 | 1,470.5 |

The median speedup in this run was **37.69×**.

Raw breakout-turbo-env rates were `293964.9`, `315810.4`, `286535.9`,
`304169.5`, and `325723.6` env-steps/s. Raw stable-retro-turbo rates were
`5475.3`, `7706.3`, `8069.2`, `9165.3`, and `8951.8` env-steps/s.

Stable Retro performs full cartridge emulation, while breakout-turbo-env is a
purpose-built behavioral reimplementation. The result measures the practical
cost difference for the matched policy path; it does not imply that the two
implementations have the same scope.
